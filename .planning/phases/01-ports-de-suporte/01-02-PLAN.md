# Plano 01-02 — Port de Saída: Módulo Clientes

## Objectivo

Criar o Value Object `DadosCliente` e a interface `ClienteRepositorioPort` que define o contrato de acesso ao módulo Clientes. O `GerirPedidoService` usará este Port para verificar que o cliente existe antes de abrir o Pedido.

## Contexto

O `GerirPedidoService` precisa de:
- Verificar que o cliente existe (buscarPorId retorna null se não existir)
- Obter o nome do cliente para associar ao Pedido

O módulo Clientes existe como módulo independente. O POS não importa directamente de `App\Modules\Clientes\Domain\Cliente` — só comunica via Port.

## Regras de Arquitectura

- **PHP puro** — sem `use Illuminate\...`, sem PDO
- **Namespace** `App\Modules\Clientes\Domain\` e `App\Modules\Clientes\Domain\Ports\Saida\`
- **Value Object simples** — `DadosCliente` contém apenas o que o POS precisa
- **Separação clara** — o `Cliente` completo (com histórico, limites de crédito, etc.) pertence ao módulo Clientes; o POS só vê `DadosCliente`

### Tarefa 1 — Criar Value Object `DadosCliente`

<read_first>
- app/Modules/POS/Domain/Pedido.php (ver o que o Pedido precisa saber sobre o cliente)
- app/Modules/Stock/Domain/DadosProduto.php (seguir o mesmo padrão de Value Object)
</read_first>

<action>
Criar o ficheiro `app/Modules/Clientes/Domain/DadosCliente.php`:

```php
<?php

namespace App\Modules\Clientes\Domain;

/**
 * DadosCliente
 *
 * Value Object que encapsula os dados de um cliente necessários ao módulo POS.
 * Imutável — representa um snapshot dos dados no momento da consulta.
 *
 * Localização: app/Modules/Clientes/Domain/DadosCliente.php
 *
 * Nota: O NIF pode ser vazio para clientes consumidor final (vendas ao balcão).
 */
final class DadosCliente
{
    public function __construct(
        private readonly int    $id,
        private readonly string $nome,
        private readonly string $nif = '',   // vazio para consumidor final
    ) {
        if (empty(trim($this->nome))) {
            throw new \DomainException('Nome do cliente não pode ser vazio');
        }
    }

    public function id(): int     { return $this->id; }
    public function nome(): string { return $this->nome; }
    public function nif(): string  { return $this->nif; }

    public function temNIF(): bool { return !empty($this->nif); }

    public function eConsumidorFinal(): bool { return !$this->temNIF(); }
}
```
</action>

```php
<?php

namespace App\Modules\Clientes\Domain\Ports\Saida;

use App\Modules\Clientes\Domain\DadosCliente;

/**
 * ClienteRepositorioPort
 *
 * Port de saída do módulo Clientes.
 * Define o contrato para consulta de dados de clientes.
 *
 * Localização: app/Modules/Clientes/Domain/Ports/Saida/ClienteRepositorioPort.php
 *
 * Implementações esperadas:
 *  - MySQLClienteRepo (Adapter Laravel — fase posterior)
 *  - FakeClienteRepositorio (para testes — Fase 2)
 */
interface ClienteRepositorioPort
{
    /**
     * Devolve os dados de um cliente pelo seu ID.
     * Retorna null se o cliente não existir.
     */
    public function buscarPorId(int $id): ?DadosCliente;
}
```
