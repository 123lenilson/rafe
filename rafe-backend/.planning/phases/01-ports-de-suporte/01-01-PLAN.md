# Plano 01-01 — Ports de Saída: Módulo Stock

## Objectivo

Criar as interfaces PHP puras que definem o contrato de acesso ao Stock. Estas interfaces serão implementadas pelos Adapters Laravel numa fase posterior. Por agora, só existem como contratos — sem lógica, sem BD.

## Contexto

O `GerirPedidoService` e o `FinalizarVendaService` precisam de:
1. Verificar se um produto existe e obter os seus dados (nome, preço, tipo, imposto)
2. Verificar se há stock suficiente antes de adicionar um item ao Pedido
3. Subtrair stock após a emissão do DocumentoFiscal

Estes três contratos pertencem ao módulo Stock. Separamos em três Ports:
- `BuscadorDeProdutoPort` — consultas de dados do produto
- `VerificadorDeStockPort` — verificação de stock disponível
- `StockRepositorioPort` — operações de movimentação de stock

## Regras de Arquitectura

- **PHP puro** — sem `use Illuminate\...`, sem PDO, sem qualquer framework
- **Namespace** `App\Modules\Stock\Domain\Ports\Saida\`
- **Retorno tipado** — todos os métodos têm tipos de retorno explícitos
- **Entidades Domain próprias** — o Port deve retornar um value object `DadosProduto`, não um array
- **Cada módulo é uma ilha** — estes Ports são os únicos pontos de entrada no módulo Stock a partir do POS

### Tarefa 1 — Criar Value Object `DadosProduto`

<read_first>
- app/Modules/POS/Domain/ItemPedido.php (ver como o Domain POS consome dados do produto — precisa de nome, preço, tipo, taxa)
- app/Modules/POS/Domain/LinhaVenda.php (ver quais campos de produto usa nos cálculos)
</read_first>

<action>
Criar o ficheiro `app/Modules/Stock/Domain/DadosProduto.php`:

```php
<?php

namespace App\Modules\Stock\Domain;

/**
 * DadosProduto
 *
 * Value Object que encapsula os dados de um produto necessários ao módulo POS.
 * Imutável — representa um snapshot dos dados no momento da consulta.
 *
 * Localização: app/Modules/Stock/Domain/DadosProduto.php
 */
final class DadosProduto
{
    public function __construct(
        private readonly int    $id,
        private readonly string $nome,
        private readonly float  $preco,
        private readonly string $tipo,    // 'P' = Produto, 'S' = Serviço
        private readonly float  $taxaImposto, // percentagem (ex: 14.0 para 14% IVA)
    ) {
        if ($this->preco < 0) {
            throw new \DomainException('Preço do produto não pode ser negativo');
        }
        if (!in_array($this->tipo, ['P', 'S'], strict: true)) {
            throw new \DomainException('Tipo de produto inválido — deve ser P (Produto) ou S (Serviço)');
        }
        if ($this->taxaImposto < 0) {
            throw new \DomainException('Taxa de imposto não pode ser negativa');
        }
    }

    public function id(): int          { return $this->id; }
    public function nome(): string      { return $this->nome; }
    public function preco(): float      { return $this->preco; }
    public function tipo(): string      { return $this->tipo; }
    public function taxaImposto(): float { return $this->taxaImposto; }

    public function eProdutoFisico(): bool { return $this->tipo === 'P'; }
    public function eServico(): bool       { return $this->tipo === 'S'; }
}
```
</action>

- Não contém `use Illuminate` nem `use PDO` nem qualquer dependência externa
