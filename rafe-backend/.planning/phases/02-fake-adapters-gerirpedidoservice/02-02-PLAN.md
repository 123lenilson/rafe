# Plano 02-02 — BDD e Testes Unitários

## Objectivo
Definir os cenários Gherkin (BDD) para a gestão de um Pedido e montar a infraestrutura de testes unitários em PHPUnit para o `GerirPedidoService`.

---

## Tarefas

### Tarefa 1 — Escrever o ficheiro BDD `gerir-pedido.feature`

<action>
Criar o ficheiro `tests/BDD/features/pos/gerir-pedido.feature`.
Escrever pelo menos os seguintes cenários (em português):
1. Iniciar um pedido para um cliente existente com sucesso.
2. Iniciar um pedido para um consumidor final (sem cliente especificado).
3. Tentar iniciar um pedido com um cliente inexistente (deve falhar).
4. Adicionar um item válido ao pedido (com stock suficiente).
5. Tentar adicionar um produto sem stock (deve falhar).
6. Tentar adicionar um produto que não existe no sistema (deve falhar).
</action>

<acceptance_criteria>
- Ficheiro `tests/BDD/features/pos/gerir-pedido.feature` existe.
- Os 6 cenários solicitados estão presentes e descritos em formato Gherkin.
</acceptance_criteria>

---

### Tarefa 2 — Scaffold de `GerirPedidoService` e respectivo Teste Unitário

<action>
1. Criar `app/Modules/POS/Services/GerirPedidoService.php` como um esboço (apenas a classe, construtor vazio, métodos vazios) para que os testes possam compilar.
   - Métodos: `iniciarPedido`, `adicionarItem`
2. Criar `tests/Unit/POS/Services/GerirPedidoServiceTest.php` cobrindo rigorosamente os cenários do Feature (Testes Red state).
   - Inserir `use PHPUnit\Framework\TestCase;`
   - O teste deve utilizar os Fake Adapters desenvolvidos no plano anterior para simular as dependências.
</action>

<acceptance_criteria>
- Ambos os ficheiros existem.
- `GerirPedidoServiceTest.php` extende de `TestCase`.
- Sem erros de sintaxe nos ficheiros (`php -l`).
</acceptance_criteria>
