# PLAN 03-03: Implementação e Infection no FinalizarVendaService

O objectivo final deste milestone: dar vida ao serviço de finalização garantindo a protecção TDD e MSI através do Infection.

## Tarefas a Executar

### 1. Implementar a Lógica no FinalizarVendaService
No método `finalizar()`, implementar rigorosamente:
1. Buscar e validar o Pedido (se não existir ou estiver vazio, lançar `DomainException`).
2. Mapear as linhas de `ItemPedido` para `LinhaVenda`.
3. Criar a instância da Entidade `DocumentoFiscal`.
4. Envolver os efeitos secundários dentro do `TransacaoPort->executar()`:
   - Fazer abate do stock iterando sobre cada item do pedido usando o `StockRepositorioPort`.
   - Obter o ID do Documento com `proximoNumeroDocumento()` e associá-lo.
   - Salvar o documento com `DocumentoFiscalRepositorioPort`.
   - Registar o pagamento via `PagamentoRepositorioPort` usando o método e o valor entregue.
   - Eliminar o pedido via `PedidoRepositorioPort`.
5. Retornar o ID do `DocumentoFiscal` gerado.

### 2. Validar Testes Unitários
- Correr os testes do `FinalizarVendaServiceTest` com PHPUnit até que todos os 4 cenários fiquem **Verde** (Pass).

### 3. Executar Mutation Testing
- Correr `vendor/bin/infection --filter=FinalizarVendaService`.
- Garantir que o Mutation Score Index (MSI) e o Covered Code MSI são superiores a 80%.

## Regras
- Nenhuma dependência estrutural do Laravel dentro do Service. Todo o comportamento transacional é assegurado através do `TransacaoPort`.
